## 11:12-11:41 GMT - Command Centre Planning & Research

### Critical Issue Identified
- **Problem**: Built duplicate HMRC UI without checking existing work
- **Root Cause**: No centralized project/task tracking system
- **Impact**: Wasted time, confusion, duplicate code
- **Solution Required**: Single source-of-truth Command Centre for ALL projects

### Deep Research Conducted

**Sources Analyzed:**
1. **Linear.app** - Developer-focused issue tracking
   - Philosophy: "Ship better, faster"
   - Key features: Minimalist UI, keyboard-first, priority levels (Urgent/High/Medium/Low)
   - Status workflow: Backlog → Todo → In Progress → Review → Done
   - Automatic progress tracking and cycle time metrics

2. **KanbanZone.com** - Visual workflow management
   - Multi-level Kanban (Portfolio → Program → Team)
   - Cumulative flow diagrams, cycle time, throughput metrics
   - Work-in-progress (WIP) limits
   - Visual bottlenecks = faster problem identification

3. **monday.com** - Project portfolio management
   - Hierarchy: Goals → Projects → Tasks → Subtasks
   - Multiple views: Kanban, Gantt, Calendar, Workload, Table
   - Dependencies, critical path, resource allocation
   - Executive visibility + team execution alignment

4. **GitLab** - DevOps integration
   - Epic → Issue → Merge Request pipeline
   - CI/CD integration, security scanning
   - Project management shouldn't be separate from dev tools

### Command Centre Design Created

**File**: `business/COMMAND-CENTRE-DESIGN.md` (8.6KB)

**Hierarchy Structure:**
```
GOALS (Strategic Objectives)
  ↓ PROJECTS (Major Initiatives)
    ↓ PHASES (Build Stages)
      ↓ TASKS (Actionable Items)
        ↓ SUBTASKS (Technical Steps)
```

**Required Views:**
1. **Dashboard** - Overview stats, today's priorities, performance metrics
2. **Kanban Board** - Drag-and-drop, swimlanes, status columns
3. **Project Detail** - Phases, timelines, dependencies
4. **Today's Focus** - In progress items, blocked items, time-sensitive

**Task Fields:**
- Priority, Status, Progress %, Time estimates vs actual
- Dependencies, Assignee (Dawn/Human/Both)
- Created/Due/Completed dates
- Checklist items
